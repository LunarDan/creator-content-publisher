# Creator Content Publisher

创作者多平台内容适配与发布助手。项目面向“多平台内容发布工具”场景：用户只需要输入一份原始内容，系统即可自动生成适合公众号、知乎、B站、小红书、抖音、快手等平台的草稿，并提供预览编辑、模拟发布、浏览器辅助发布、公众号草稿发布和发布历史管理能力，帮助创作者减少重复排版与多平台分发成本。

## 项目定位

很多创作者需要在多个内容平台同步发布相同主题的内容，但不同平台对标题长度、正文风格、标签、视频文件、封面、创作声明、内容声明等要求不同。本项目通过“统一内容输入 + 平台适配器 + 发布任务管理”的方式，提高内容分发效率。

项目当前采用 Web-first 形态，功能已覆盖完整的内容发布闭环：

- 前端：Vue 3 + Vite + Element Plus 2（Warm Teal 设计系统 + CSS Token 体系）
- 后端：Flask + SQLite
- 发布方式：模拟发布、公众号草稿发布、多平台浏览器辅助真实发布

> 说明：对于知乎、小红书、快手等存在登录、验证码、安全验证或平台风控的平台，系统采用保守合规的浏览器发布助手模式，只辅助打开页面、上传/填充内容，不绕过平台风控，也不强制自动点击最终发布按钮。

## 已实现功能

### 1. 内容创作

- 输入原始标题、摘要、正文
- 配置封面地址或本地路径
- 配置视频文件路径
- 配置标签
- 选择目标发布平台

### 2. 多平台自动适配

系统会根据不同平台特点生成对应草稿，目前支持：

| 平台 | 适配能力 |
| --- | --- |
| 公众号 | 长文排版、摘要、结构化表达 |
| 知乎 | 理性分析风格、标题与正文适配、话题标签、文章封面、创作声明 |
| B站 | 视频简介、标题、标签、创作声明配置 |
| 小红书 | 短标题、种草文案、分段、话题标签、视频路径、封面、内容声明、原创声明 |
| 抖音 | 短视频标题、简介、话题标签、视频路径 |
| 快手 | 短视频描述、话题标签、视频路径、封面和作者声明 |

### 3. 草稿预览与编辑

- 查看不同平台的适配结果
- 修改标题、摘要、正文和标签
- 平台专属配置：
  - B站：视频文件路径、创作声明
  - 知乎：文章封面、创作声明
  - 小红书：视频文件路径、封面图片路径、内容声明、原创声明
  - 快手：视频文件路径、封面图片路径、作者声明/补充说明

### 4. 发布中心

- 一键模拟发布多个平台草稿
- 一键启动多平台真实发布流程
- 单平台发布按钮：
  - 保存到公众号草稿箱
  - 浏览器打开知乎发布助手
  - 浏览器打开小红书发布助手
  - 浏览器辅助发布到抖音
  - 浏览器保存到 B站草稿
  - 浏览器打开快手发布助手

### 5. 发布历史

- 查看模拟发布和真实发布任务
- 查看平台、标题、发布模式、状态、链接、外部 ID 等信息
- 对浏览器辅助发布任务支持人工确认完成

### 6. 平台配置说明

设置页提供各平台发布能力说明：

- 公众号草稿发布配置检测
- B站浏览器发布说明
- 知乎发布助手说明
- 小红书浏览器发布助手说明
- 抖音浏览器辅助发布说明
- 快手浏览器发布助手说明

## 技术架构

```text
frontend/                  Vue 3 + Vite 前端
  src/api/                 API 请求封装
  src/components/          通用组件
  src/router/              前端路由
  src/stores/              Pinia 状态管理
  src/styles/              全局样式
  src/utils/               工具函数
  src/views/               页面视图

backend/                   Flask + SQLite 后端
  adapters/                平台内容适配器
  publishers/              平台发布器 / 浏览器发布助手
  repositories/            SQLite 数据访问层
  routes/                  HTTP API 路由
  services/                业务编排服务
  init_db.py               数据库初始化
  app.py                   Flask 入口
```

### 后端分层设计

| 层级 | 职责 |
| --- | --- |
| routes | 暴露 HTTP API |
| services | 编排内容、草稿、发布任务等业务流程 |
| repositories | 负责 SQLite 数据读写 |
| adapters | 将原始内容转换为平台草稿 |
| publishers | 负责真实发布、草稿发布或浏览器辅助发布 |

## 扩展更多平台的设计

项目采用 Adapter + Publisher 的扩展方式，新增平台时不需要重写核心流程。

### 1. 新增平台适配器

在 `backend/adapters/` 中新增平台适配器，例如：

```python
from .base_adapter import BaseAdapter


class NewPlatformAdapter(BaseAdapter):
    platform_key = 'new_platform'
    platform_name = '新平台'
    description = '适合某类内容分发。'

    def transform(self, content):
        return {
            'platform': self.platform_key,
            'title': self._trim_title(content['title'], 30),
            'summary': content.get('summary', ''),
            'body': content['body'],
            'tags': self._base_tags(content, 5),
            'cover_image': content.get('cover_image', ''),
            'extra_config': {},
            'validation_warnings': [],
        }
```

然后在 `backend/adapters/registry.py` 中注册该适配器。

### 2. 新增平台发布器

如果需要真实发布能力，可在 `backend/publishers/` 中新增发布器：

- API 发布：适合平台开放接口完整、鉴权稳定的场景
- 浏览器辅助发布：适合没有开放 API 或需要人工确认的平台
- 模拟发布：适合课程演示或早期开发验证

浏览器发布器通常遵循以下结构：

```text
backend/publishers/<platform>_browser.py
  publish(draft, auto_publish=False)
  打开平台创作页
  上传视频/图片
  填充标题、正文、标签等字段
  返回 manual_pending，等待用户人工确认最终发布
```

### 3. 新增发布接口和前端入口

- 在 `backend/routes/publish.py` 中增加发布接口
- 在 `backend/services/content_service.py` 中增加业务编排逻辑
- 在 `frontend/src/api/publish.js` 中增加 API 封装
- 在发布中心增加对应按钮或批量发布逻辑

## 快速开始

### 环境要求

- Python 3.10+
- Node.js 18+
- npm
- Chromium / Playwright 浏览器环境，真实浏览器辅助发布时需要

### 1. 安装后端依赖

```bash
cd backend
pip install -r requirements.txt
```

如果需要浏览器辅助发布，请安装 Playwright 浏览器：

```bash
python -m playwright install chromium
```

### 2. 启动后端

在项目根目录执行：

```bash
py -m backend.app
```

默认地址：

```text
http://127.0.0.1:5409
```

健康检查：

```text
http://127.0.0.1:5409/api/health
```

### 3. 安装前端依赖

```bash
cd frontend
npm install
```

### 4. 启动前端

```bash
npm run dev
```

默认地址通常是：

```text
http://127.0.0.1:5173
```

如果端口被占用，Vite 会自动切换到其他端口，例如 `5174`、`5175`、`5177`、`5178`。

## 使用流程

1. 打开前端页面
2. 在“内容创作”中输入原始内容
3. 选择目标平台
4. 点击“保存并生成平台草稿”
5. 在“预览中心”检查和微调各平台草稿
6. 在“发布中心”执行：
   - 一键模拟发布
   - 或启动对应平台发布助手
7. 在“发布历史”查看发布任务状态
8. 对需要人工确认的平台，完成后点击确认记录

## 真实发布说明

### 公众号

支持通过公众号接口创建草稿。需要配置：

- `WECHAT_APP_ID`
- `WECHAT_APP_SECRET`
- `WECHAT_DEFAULT_THUMB_MEDIA_ID`

可参考 `.env.example` 创建 `.env` 文件。

### 知乎

采用保守合规的浏览器发布助手：

- 打开知乎创作页
- 尝试填充标题、正文、话题
- 尝试设置文章封面和创作声明
- 不自动点击最终发布按钮
- 遇到登录、验证码、安全验证或风控时停下等待人工处理

### 小红书

采用保守合规的浏览器发布助手，发布逻辑参考 `social-auto-upload-web-ui` 项目的小红书真实发布流程，并适配到当前项目架构：

- 打开小红书创作者发布页
- 使用本地持久化浏览器目录保存登录态
- 尝试上传视频文件
- 等待视频上传或处理完成
- 填写 20 字以内标题
- 填写正文描述
- 添加话题标签
- 尝试设置封面图片
- 尝试设置内容声明
- 尝试设置原创声明
- 不自动点击最终发布按钮
- 遇到登录、验证码、安全验证或风控时停下等待人工处理

小红书浏览器登录态默认存放在：

```text
backend/data/browser/xiaohongshu
```

### B站

支持浏览器辅助发布：

- 打开 B站创作中心
- 尝试上传视频
- 填充标题和简介
- 尝试保存草稿或等待人工确认

### 抖音

支持浏览器辅助发布：

- 打开抖音创作者中心
- 尝试上传视频
- 填充标题、简介和话题标签
- 遇到验证码、安全验证或风控时等待人工处理

### 快手

支持浏览器发布助手：

- 打开快手创作页
- 尝试上传视频
- 填充描述和标签
- 尝试设置封面和作者声明
- 不自动点击最终发布按钮
- 支持关闭浏览器后重新唤起发布助手

## 安全与合规说明

本项目的浏览器辅助发布功能仅用于提高创作者发布效率，不用于绕过平台规则。对于需要登录、验证码、安全验证、实名认证、手机号验证或平台风控的情况，系统会停下并交由用户人工处理。

项目不会保存平台账号密码，也不会提供绕过验证码、规避风控或批量滥用平台发布能力的功能。知乎、小红书、快手等平台默认采用人工确认最终发布的方式，以保证内容发布前由用户检查确认。
